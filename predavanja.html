Sidrenje i povezivanje

Ovo predavanje se bavi temom povezivanja sadržaja i dokumenata unutar web stranica. Kada organiziramo web sjedište često sadržaj spremamo u mape poredane hijerarhijski u više razina, kako bi dokumenti bili odijeljeni u logičke cjeline. Npr. slike koje koristimo u stranicama spremit ćemo u mapu slike, druge dokumente u drugu mapu ili podmapu itd... Što je web sjedište složenije - važnost dobro organiziranog sadržaja je sve veća zbog lakšeg snalaženja i povezivanja.

Html dokumenti koji čine jedno web sjedište su uvijek povezani linkovima (odn. sidrima - eng. anchor), a način na koji ih povezujemo ovisi o njihovoj poziciji unutar različitih razina mapa i podmapa.

Razlikujemo

- povezivanje između različitih dokumenata na različitim razinama - relativno povezivanje

- povezivanje sadržaja unutar jednog ili više dokumenata - sidrenje unutar stranice

2. SIDRENJE UNUTAR STRANICA

Ovdje se povezuje sadržaj unutar istog ili drugog dokumenta. Taj sadržaj će se prikazati na vrhu stranice.

Da bi se moglo povezati na točno određeni sadržaj, potrebno ga je imenovati atributima name ili id te u poveznici navesti ime elementa na koji se povezujemo.

U priloženim primjerima prvi dokument sadrži samo linkove na sadržaj, a drugi dokument sadrži tekst opisa pojedinog linka.

sadržaj označavamo imenom npr.

<"p name = "prvi_paragraf">ovo je tekst na koji se povezujem ili <"p id = "prvi_paragraf">ovo je tekst na koji se povezujem a na njega se povezujemo tako da u linku uz ime dokumenta u kojem se nalazi sadržaj dodamo i ime tog sadržaja sa znakom #. Atribut name se može umetnuti u bilo koji html element na koji se želimo povezati (usidriti).
